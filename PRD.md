## 1. 产品概述

### 1.1 一句话定义
MetaCraft Agent 是一款面向材料加工产线（热处理 / 焊接 / 轧制等）的智能工艺优化 Agent，能自主完成"缺陷归因 → 工艺参数推荐 → 操作员确认 → 效果跟踪"的闭环任务，把资深工艺工程师的经验沉淀为可执行系统。

### 1.2 产品定位
- **不是**：替代工艺工程师的黑盒系统
- **是**：工艺工程师 + 操作员的"AI 副手"，把人从重复归因劳动中解放出来，让经验沉淀并复用

### 1.3 差异化优势
| 维度 | 现有 MES / SPC | 通用 AI 助手 | MetaCraft Agent |
|------|----------------|--------------|-----------------|
| 工艺领域知识 | 弱（仅数据展示） | 浅（无专业沉淀） | 强（机理 + 案例 RAG） |
| 自主归因能力 | 无 | 弱 | 强（任务拆解 + 多 Agent） |
| 记忆与经验积累 | 无 | 短期 | 三层记忆架构 |
| 人机协作设计 | 无 | 通用 | 心理学优化交互 |
| 可解释性 | 高 | 低 | 高（每步可追溯） |

## 2. 用户画像

### 2.1 核心用户
| 角色 | 占比 | 核心诉求 | 痛点 |
|------|------|---------|------|
| **操作员** | 60% | 快速拿到参数调整建议 | 缺陷出现时不知如何归因，等工艺员响应慢 |
| **工艺工程师** | 25% | 批量分析历史缺陷、沉淀经验 | 重复归因劳动，经验难复用 |
| **质量主管** | 10% | 全厂缺陷率趋势、根因分布 | 看不到全貌，决策靠拍脑袋 |
| **新员工** | 5% | 快速学习工艺规则 | 老师傅不愿教、手册太抽象 |

### 2.2 反向用户（暂不服务）
- 纯设备维护人员（设备故障诊断走另一套系统）
- 跨工厂管理决策层（需要 BI 而非 Agent）

## 3. 用户痛点与场景

### 3.1 三大核心痛点

**痛点 1：缺陷归因靠经验、靠试错**
- 现状：硬度偏低 → 老师傅凭经验猜"保温时间不够"，新手完全没头绪
- 影响：归因时间长（平均 4 小时），错误归因导致返工率上升

**痛点 2：经验沉淀难、流失快**
- 现状：老师傅退休，经验跟着走；新人靠手册，手册更新滞后
- 影响：工艺知识断层，新产线调试周期长

**痛点 3：参数调整无闭环**
- 现状：调参后没有系统跟踪效果，无法验证"调对没调对"
- 影响：相同缺陷反复出现，无法持续改进

### 3.2 典型场景剧本

**场景 A：批次硬度异常归因**
> 操作员小王在检测工位发现批次 #20260628-A 硬度偏低 5 HRc。
> 他打开 MetaCraft Agent 输入："批次 #20260628-A 硬度偏低"。
> Agent 在 90 秒内返回：
> 1. 可能原因排序（保温时间不足 65%、淬火冷却速率不足 20%、原材料偏差 15%）
> 2. 每个原因的证据链（实际参数 vs 标准窗口）
> 3. 推荐调整方案（保温时间 +15 分钟）
> 4. 相似历史案例 3 条
> 小王确认后，Agent 提交参数调整建议给工艺工程师审核。

**场景 B：工艺知识沉淀**
> 工艺工程师老李完成一次复杂归因后，Agent 主动询问："是否将本次案例沉淀为知识？"
> 老李确认，Agent 自动结构化存储：缺陷类型 / 工艺参数 / 根因 / 解决方案 / 效果，进入长期记忆。

## 4. 产品目标

### 4.1 北极星指标
**单位产线月度缺陷归因平均耗时（目标：4h → 0.5h）**

### 4.2 阶段性目标
| 阶段 | 时间 | 核心目标 | 量化指标 |
|------|------|---------|---------|
| MVP | 3 个月 | 单产线跑通闭环 | 归因准确率 ≥ 70%，响应 ≤ 5min |
| V1 | 6 个月 | 多产线部署 | 覆盖 3 条产线，采纳率 ≥ 60% |
| V2 | 12 个月 | 经验自进化 | 缺陷率下降 ≥ 15%，月沉淀案例 ≥ 100 条 |

## 5. 功能需求

### 5.1 功能架构总览

```
MetaCraft Agent
├── F1 任务接收与拆解
├── F2 自主归因执行
├── F3 工艺参数推荐
├── F4 记忆管理
├── F5 多 Agent 协作
├── F6 结果评估与反馈
├── F7 人机交互界面
└── F8 知识沉淀与管理
```

### 5.2 详细功能需求

#### F1. 任务接收与拆解
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F1.1 | 自然语言输入 | 支持文本/语音输入缺陷描述 | P0 |
| F1.2 | 批次 ID 自动取数 | 输入批次号自动拉取工艺参数 | P0 |
| F1.3 | 任务自动拆解 | 把"硬度偏低"拆成 3-5 个子任务 | P0 |
| F1.4 | 拆解结果可视化 | 展示子任务列表与执行顺序 | P1 |
| F1.5 | 用户可干预拆解 | 用户可手动修改子任务 | P2 |

**F1 验收标准**：
- 输入"批次 X 硬度偏低 Y HRc"，5 秒内输出拆解后的子任务列表
- 拆解覆盖：参数查询、标准对比、历史检索、机理验证、综合建议

#### F2. 自主归因执行
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F2.1 | 子任务串/并行执行 | 支持并行调用数据查询与知识检索 | P0 |
| F2.2 | 工具调用 | 调用 MES 查询、机理模型、RAG 检索 | P0 |
| F2.3 | 中间结果反思 | 每步执行后判断是否需要重新规划 | P0 |
| F2.4 | 执行过程透明化 | 实时展示当前执行步骤与中间结果 | P1 |
| F2.5 | 异常处理 | 工具调用失败时的降级与重试 | P1 |

**F2 验收标准**：
- 完整归因流程在 5 分钟内完成
- 单工具失败不阻断整体流程，自动降级
- 每个子任务的输入/输出/耗时可追溯

#### F3. 工艺参数推荐
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F3.1 | 多候选方案生成 | 至少给出 2 个调整方案 | P0 |
| F3.2 | 方案证据链 | 每个方案标注依据（机理/案例/经验） | P0 |
| F3.3 | 风险提示 | 标注方案对其他性能指标的影响 | P0 |
| F3.4 | 方案对比 | 表格化对比方案优劣 | P1 |
| F3.5 | 方案执行接口 | 提交至工艺审核流程 | P1 |

**F3 验收标准**：
- 每个方案含：调整项、调整量、预期效果、风险、依据来源
- 不输出违反工艺约束的方案（如超过设备上限）

#### F4. 记忆管理
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F4.1 | 工作记忆 | 当前会话的批次上下文 | P0 |
| F4.2 | 短期记忆 | 近 30 天批次与归因记录 | P0 |
| F4.3 | 长期语义记忆 | 工艺手册、历史案例的向量检索 | P0 |
| F4.4 | 记忆权重更新 | 根据用户反馈调整检索权重 | P1 |
| F4.5 | 记忆可视化 | 展示当前 Agent "记得"什么 | P2 |

**F4 验收标准**：
- 检索相似案例 Top-3 准确率 ≥ 80%
- 记忆写入到可用 < 10 秒
- 工作记忆不污染长期记忆（隔离）

#### F5. 多 Agent 协作
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F5.1 | 角色定义 | 数据/机理/知识/决策/审核/交互 6 个角色 | P0 |
| F5.2 | 协作流程编排 | LangGraph 定义节点与边 | P0 |
| F5.3 | 并行调度 | 独立子任务并行执行 | P1 |
| F5.4 | 冲突仲裁 | 多 Agent 结论冲突时的仲裁机制 | P1 |
| F5.5 | 协作过程可视化 | 展示 Agent 间消息流 | P2 |

**F5 验收标准**：
- 6 个角色 Agent 各自职责清晰，无越权
- 协作流程可配置（不同场景不同流程）

#### F6. 结果评估与反馈
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F6.1 | LLM 自评 | 审核 Agent 评估输出合理性 | P0 |
| F6.2 | 用户反馈采集 | 操作员对建议打分（采纳/拒绝/部分采纳） | P0 |
| F6.3 | 效果跟踪 | 调参后批次质量自动跟踪 | P1 |
| F6.4 | 评估指标看板 | 准确率/采纳率/响应时间可视化 | P1 |
| F6.5 | 失败案例归集 | 自动收集低分案例用于改进 | P1 |

**F6 验收标准**：
- 每条建议都有反馈通道
- 效果跟踪 T+7 天自动出报告

#### F7. 人机交互界面
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F7.1 | 对话主界面 | 类 ChatGPT 交互 | P0 |
| F7.2 | 推荐卡片 | 参数建议以卡片形式展示 | P0 |
| F7.3 | 一键确认/拒绝 | 操作员快速反馈 | P0 |
| F7.4 | 工艺上下文侧栏 | 展示当前批次参数与历史 | P1 |
| F7.5 | 语音输入 | 解放操作员双手 | P2 |
| F7.6 | 信任度展示 | Agent 标注自身不确定度 | P1 |

**F7 设计原则（心理学结合点）**：
- **可控感**：用户随时可干预，不是黑盒
- **可解释性**：每个建议都有依据
- **渐进披露**：先结论，后展开证据
- **不确定度透明**：明确标注置信度，避免过度信任

#### F8. 知识沉淀与管理
| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|

| F8.2 | 案例库管理 | 增删改查、分类标签 | P1 |
| F8.3 | 工艺手册导入 | PDF/Word 文档向量化入库 | P1 |
| F8.4 | 知识冲突检测 | 新知识与旧知识矛盾时告警 | P2 |
| F8.5 | 知识导出 | 导出为标准工艺文档 | P2 |

## 6. Agent 七大要素设计

### 6.1 自主行动
- **机制**：LangGraph 状态机 + 条件边 + 循环
- **边界**：自主执行归因与建议生成；**不自主**直接修改产线参数（必须经人工确认）

### 6.2 任务拆解
- **机制**：Plan-and-Execute 模式
- **拆解原则**：每个子任务可独立验证、可重试、可降级
- **示例**：
  ```
  原任务：批次 #A 硬度偏低
  拆解：
    1. query_batch_params(#A)         [工具调用]
    2. compare_with_standard(params)  [工具调用]
    3. search_similar_cases("硬度偏低") [向量检索]
    4. run_metallurgy_model(params)    [模型调用]
    5. synthesize_report(以上结果)     [LLM 综合]
  ```

### 6.3 工具调用
- **协议**：MCP（Model Context Protocol）
- **工具清单**：
  | 工具名 | 功能 | 数据源 |
  |--------|------|--------|
  | query_batch_params | 查批次工艺参数 | MES / SCADA |
  | query_defect_history | 查历史缺陷 | 缺陷库 |
  | run_metallurgy_model | 调机理模型 | Python 模型 |
  | search_handbook | 检索工艺手册 | 向量库 |
  | search_cases | 检索历史案例 | 向量库 |
  | submit_adjustment | 提交参数调整 | MES API |

### 6.4 记忆管理
- **三层架构**：
  ```
  ┌─────────────────────────────┐
  │ 工作记忆 (Working)          │  ← LangGraph State
  │  当前会话上下文             │
  ├─────────────────────────────┤
  │ 短期记忆 (Episodic)         │  ← SQLite + 时间索引
  │  近 30 天批次与归因         │
  ├─────────────────────────────┤
  │ 长期记忆 (Semantic)         │  ← Chroma 向量库
  │  工艺手册 + 历史案例        │
  └─────────────────────────────┘
  ```
- **更新机制**：
  - 工作记忆：每次对话自动更新
  - 短期记忆：归因完成后自动写入
  - 长期记忆：用户确认后写入，含质量评分

### 6.5 角色设定
| Agent 角色 | Prompt 要点 | 心理学结合 |
|-----------|-------------|-----------|
| 数据 Agent | 严谨、只报数据、不做推断 | 类似"数据型人格" |
| 机理 Agent | 基于物理冶金原理，可证伪 | 类似"分析型专家" |
| 知识 Agent | 引用来源、不臆造 | 类似"图书管理员" |
| 决策 Agent | 权衡多目标、给排序 | 类似"经验型师傅" |
| 审核 Agent | 挑刺、质疑、保守 | 类似"质监员" |
| 交互 Agent | 用户友好、说人话 | 共情式表达 |

### 6.6 协作流程
```
用户输入
    ↓
[任务拆解]
    ↓
[数据 Agent] ──→ 参数
    ↓
[机理 Agent] ─┐
[知识 Agent] ─┤ (并行)
    ↓         ↓
[决策 Agent] ←─┘
    ↓
[审核 Agent] ──(不通过)──→ [决策 Agent] 重做
    ↓ (通过)
[交互 Agent] ──→ 用户
    ↓
[用户反馈] ──→ [记忆管理]
```

### 6.7 结果评估
- **三层评估**：
  1. **任务级**：归因准确率、采纳率、响应时间
  2. **LLM 自评**：审核 Agent + 另一 LLM 双重评估
  3. **业务级**：缺陷率下降、良品率提升、操作员信任度问卷
